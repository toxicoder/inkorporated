# Logging and Monitoring

## Logging and Monitoring

- Use structured logging formats (e.g., JSON)
- Log errors with stack traces and context
- Implement log levels: debug, info, warn, error
- Integrate with monitoring tools for alerts
- Avoid logging sensitive information
- Rotate and archive logs regularly
- Monitor key metrics like response times and error rates
- Use distributed tracing
- Set up dashboards for metrics
- Log user actions for auditing
- Implement log aggregation
- Alert on anomalies
- Use sampling for high-volume logs
- Document logging conventions
- Integrate with error tracking services
- Monitor resource usage
- Test logging in production-like env
- **Use MCP tools for enhanced logging and monitoring capabilities**
- **Integrate MCP-based analytics for better observability**
