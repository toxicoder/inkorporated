# Code Style & Patterns

## Code Style & Patterns

- Generate API clients using OpenAPI Generator
- Use TypeScript axios template
- Place generated code in /src/generated
- Prefer composition over inheritance
- Use repository pattern for data access
- Enforce consistent naming conventions: camelCase for variables, PascalCase for classes
- Limit function length to 100 lines maximum
- Use async/await over callbacks for asynchronous code
- Implement dependency injection for testable code
- Avoid global variables; use modules for shared state
- Use type annotations or interfaces for function parameters and returns
- Prefer immutable data structures
- Use guards for early function exits
- Follow SOLID principles
- Use factory patterns for object creation
- Implement observer pattern for event handling
- Use decorators for cross-cutting concerns
- Avoid deep nesting; flatten logic where possible
- Use enums for constant values
- Prefer arrow functions for callbacks
- Organize code into feature folders
