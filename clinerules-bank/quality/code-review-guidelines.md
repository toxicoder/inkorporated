# Code Review Guidelines

## Code Review Guidelines

- Require at least one approval for merges
- Keep pull requests small and focused
- Provide constructive feedback in reviews
- Check for adherence to guidelines in reviews
- Respond to review comments within 24 hours
- Include screenshots or demos for UI changes
- Ensure tests pass before requesting review
- Use checklists in PR templates
- Assign reviewers based on expertise
- Avoid nitpicking; focus on important issues
- Encourage pair programming for complex changes
- Document review decisions
- Use automated tools for style checks
- Require sign-off for security changes
- Limit review scope to changes only
- Celebrate good PRs with thanks
- Track review metrics for improvement
